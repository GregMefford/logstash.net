Synopsis

This is what it might look like in your config file: 

output {
  statsd {
    count => ... # hash, default: {}
    debug => ... # boolean
    decrement => ... # array, default: []
    host => ... # string, default: "localhost"
    increment => ... # array, default: []
    namespace => ... # string, default: "logstash"
    port => ... # number, default: 8125
    sample_rate => ... # number, default: 1
    sender => ... # string, default: "%{@source_host}"
    tags => ... # array, default: []
    timing => ... # hash, default: {}
  }
}

Details

count

  • Value type is hash
  • Default value is {}

A count metric. metric_name => count as hash

debug

  • Value type is boolean
  • There is no default value for this setting.

The final metric sent to statsd will look like the following (assuming defaults) logstash.sender.file_name

Enable debugging output?

decrement

  • Value type is array
  • Default value is []

A decrement metric. metric names as array.

host

  • Value type is string
  • Default value is "localhost"

Regex stolen from statsd code The address of the Statsd server.

increment

  • Value type is array
  • Default value is []

An increment metric. metric names as array.

namespace

  • Value type is string
  • Default value is "logstash"

The statsd namespace to use for this metric

port

  • Value type is number
  • Default value is 8125

The port to connect to on your statsd server.

sample_rate

  • Value type is number
  • Default value is 1

The sample rate for the metric

sender

  • Value type is string
  • Default value is "%{@source_host}"

The name of the sender. Dots will be replaced with underscores

tags

  • Value type is array
  • Default value is []

Only handle these tagged events Optional.

timing

  • Value type is hash
  • Default value is {}

A timing metric. metric_name => duration as hash
