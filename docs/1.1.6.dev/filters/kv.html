kv

Status: beta

This filter helps automatically parse messages which are of the 'foo=bar' variety.

For example, if you have a log message which contains 'ip=1.2.3.4 error=REFUSED', you can parse those automatically by doing:

filter {

 kv { }

}

And you will get field 'ip' == "1.2.3.4" etc.

Synopsis

This is what it might look like in your config file: 
filter {
  kv {
    add_field => ... # hash (optional), default: {}
    add_tag => ... # array (optional), default: []
    container => ... # string (optional), default: "@fields"
    exclude_tags => ... # array (optional), default: []
    field_split => ... # string (optional), default: " "
    fields => ... # array (optional), default: ["@message"]
    prefix => ... # string (optional), default: ""
    remove_tag => ... # array (optional), default: []
    tags => ... # array (optional), default: []
    trim => ... # string (optional)
    type => ... # string (optional), default: ""
    value_split => ... # string (optional), default: "="
  }
}

Details

add_field

  • Value type is hash
  • Default value is {}

add_tag

  • Value type is array
  • Default value is []

container

  • Value type is string
  • Default value is "@fields"

exclude_tags

  • Value type is array
  • Default value is []

field_split

  • Value type is string
  • Default value is " "

fields

  • Value type is array
  • Default value is ["@message"]

prefix

  • Value type is string
  • Default value is ""

remove_tag

  • Value type is array
  • Default value is []

tags

  • Value type is array
  • Default value is []

trim

  • Value type is string
  • There is no default value for this setting.

type

  • Value type is string
  • Default value is ""

value_split

  • Value type is string
  • Default value is "="
