xml

Status: experimental

XML filter. Takes a field that contains XML and expands it into an actual datastructure.

Synopsis

This is what it might look like in your config file: 
filter {
  xml {
    /[A-Za-z0-9_-]+/ => ... # string (optional)
    add_field => ... # hash (optional), default: {}
    add_tag => ... # array (optional), default: []
    exclude_tags => ... # array (optional), default: []
    remove_tag => ... # array (optional), default: []
    store_xml => ... # boolean (optional), default: true
    tags => ... # array (optional), default: []
    type => ... # string (optional), default: ""
    xpath => ... # hash (optional), default: {}
  }
}

Details

/[A-Za-z0-9_-]+/

  • The configuration attribute name here is anything that matches the above regular expression.
  • Value type is string
  • There is no default value for this setting.

add_field

  • Value type is hash
  • Default value is {}

add_tag

  • Value type is array
  • Default value is []

exclude_tags

  • Value type is array
  • Default value is []

remove_tag

  • Value type is array
  • Default value is []

store_xml

  • Value type is boolean
  • Default value is true

tags

  • Value type is array
  • Default value is []

type

  • Value type is string
  • Default value is ""

xpath

  • Value type is hash
  • Default value is {}
