syslog

Status: experimental

Send events to a syslog server.

You can send messages compliant with RFC3164 or RFC5424 UDP or TCP syslog transport is supported

Synopsis

This is what it might look like in your config file: 
output {
  syslog {
    appname => ... # string (optional), default: "LOGSTASH"
    exclude_tags => ... # array (optional), default: []
    facility => ... # ["kernel", "user-level", "mail", "daemon", "security/authorization", "syslogd", "line printer", "network news", "uucp", "clock", "security/authorization", "ftp", "ntp", "log audit", "log alert", "clock", "local0", "local1", "local2", "local3", "local4", "local5", "local6", "local7"] (required)
    fields => ... # array (optional), default: []
    host => ... # string (required)
    msgid => ... # string (optional), default: "-"
    port => ... # number (required)
    procid => ... # string (optional), default: "-"
    protocol => ... # ["tcp", "udp"] (optional), default: "udp"
    rfc => ... # ["rfc3164", "rfc5424"] (optional), default: "rfc3164"
    severity => ... # ["emergency", "alert", "critical", "error", "warning", "notice", "informational", "debug"] (required)
    sourcehost => ... # string (optional), default: "%{@source_host}"
    tags => ... # array (optional), default: []
    timestamp => ... # string (optional), default: "%{@timestamp}"
    type => ... # string (optional), default: ""
  }
}

Details

appname

  • Value type is string
  • Default value is "LOGSTASH"

exclude_tags

  • Value type is array
  • Default value is []

facility (required setting)

  • Value can be any of: "kernel", "user-level", "mail", "daemon", "security/authorization", "syslogd", "line printer", "network news", "uucp", "clock", "security/authorization", "ftp", "ntp", "log audit", "log alert", "clock", "local0", "local1", "local2", "local3", "local4", "local5", "local6", "local7"
  • There is no default value for this setting.

fields

  • Value type is array
  • Default value is []

host (required setting)

  • Value type is string
  • There is no default value for this setting.

msgid

  • Value type is string
  • Default value is "-"

port (required setting)

  • Value type is number
  • There is no default value for this setting.

  • Default value is "udp"

rfc

severity (required setting)
