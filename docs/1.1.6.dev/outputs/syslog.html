syslog

Status: experimental

Send events to a syslog server.

You can send messages compliant with RFC3164 or RFC5424 UDP or TCP syslog transport is supported

Synopsis

This is what it might look like in your config file: 
output {
  syslog {
    appname => ... #  (optional)
    exclude_tags => ... #  (optional)
    facility => ... #  (optional)
    fields => ... #  (optional)
    host => ... #  (optional)
    msgid => ... #  (optional)
    port => ... #  (optional)
    procid => ... #  (optional)
    protocol => ... #  (optional)
    rfc => ... #  (optional)
    severity => ... #  (optional)
    sourcehost => ... #  (optional)
    tags => ... #  (optional)
    timestamp => ... #  (optional)
    type => ... #  (optional)
  }
}

Details

appname

  • Value type is string
  • There is no default value for this setting.

application name for syslog message

exclude_tags

  • Value type is string
  • There is no default value for this setting.

Only handle events without any of these tags. Note this check is additional to type and tags.

facility

  • Value type is string
  • There is no default value for this setting.

facility label for syslog message

fields

  • Value type is string
  • There is no default value for this setting.

Only handle events with all of these fields. Optional.

host

  • Value type is string
  • There is no default value for this setting.

syslog server address to connect to

msgid

  • Value type is string
  • There is no default value for this setting.

message id for syslog message

port

  • Value type is string
  • There is no default value for this setting.

syslog server port to connect to

procid

  • Value type is string
  • There is no default value for this setting.

process id for syslog message

protocol

  • Value type is string
  • There is no default value for this setting.

syslog server protocol. you can choose between udp and tcp

rfc

  • Value type is string
  • There is no default value for this setting.

syslog message format: you can choose between rfc3164 or rfc5424

severity

  • Value type is string
  • There is no default value for this setting.

severity label for syslog message

sourcehost

  • Value type is string
  • There is no default value for this setting.

source host for syslog message

tags

  • Value type is string
  • There is no default value for this setting.

Only handle events with all of these tags. Note that if you specify a type, the event must also match that type. Optional.

timestamp

  • Value type is string
  • There is no default value for this setting.

timestamp for syslog message

type

  • Value type is string
  • There is no default value for this setting.

The type to act on. If a type is given, then this output will only act on messages with the same type. See any input plugin's "type" attribute for more. Optional.
