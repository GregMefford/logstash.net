circonus

Status: experimental

Synopsis

This is what it might look like in your config file: 
output {
  circonus {
    annotation => ... # hash (required), default: {}
    annotation => ... #  (optional)
    api_token => ... # string (required)
    api_token => ... #  (optional)
    app_name => ... #  (optional)
    app_name => ... # string (required)
    exclude_tags => ... #  (optional)
    exclude_tags => ... # array (optional), default: []
    fields => ... # array (optional), default: []
    fields => ... #  (optional)
    tags => ... # array (optional), default: []
    tags => ... #  (optional)
    type => ... # string (optional), default: ""
    type => ... #  (optional)
  }
}

Details

annotation (required setting)

  • Value type is hash
  • Default value is {}

annotation

  • Value type is string
  • There is no default value for this setting.

Annotations Registers an annotation with Circonus The only required field is title and description. start and stop will be set to event.unix_timestamp You can add any other optional annotation values as well. All values will be passed through event.sprintf

Example: ["title":"Logstash event", "description":"Logstash event for %{@sourcehost}"] or ["title":"Logstash event", "description":"Logstash event for %{@sourcehost}", "parent_id", "1"]

api_token (required setting)

  • Value type is string
  • There is no default value for this setting.

api_token

  • Value type is string
  • There is no default value for this setting.

This output lets you send annotations to Circonus based on Logstash events

Your Circonus API Token

app_name

  • Value type is string
  • There is no default value for this setting.

Your Circonus App name This will be passed through event.sprintf so variables are allowed here:

Example: app_name => "%{myappname}"

app_name (required setting)

  • Value type is string
  • There is no default value for this setting.

exclude_tags

  • Value type is string
  • There is no default value for this setting.

Only handle events without any of these tags. Note this check is additional to type and tags.

exclude_tags

  • Value type is array
  • Default value is []

fields

  • Value type is array
  • Default value is []

fields

  • Value type is string
  • There is no default value for this setting.

Only handle events with all of these fields. Optional.

tags

  • Value type is array
  • Default value is []

tags

  • Value type is string
  • There is no default value for this setting.

Only handle events with all of these tags. Note that if you specify a type, the event must also match that type. Optional.

type

  • Value type is string
  • Default value is ""

type

  • Value type is string
  • There is no default value for this setting.

The type to act on. If a type is given, then this output will only act on messages with the same type. See any input plugin's "type" attribute for more. Optional.
