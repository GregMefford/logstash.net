datadog

Status: experimental

Synopsis

This is what it might look like in your config file: 
output {
  datadog {
    alert_type => ... # ["info", "error", "warning", "success"] (optional)
    api_key => ... # string (required)
    date_happened => ... # string (optional)
    dd_tags => ... # array (optional)
    exclude_tags => ... # array (optional), default: []
    fields => ... # array (optional), default: []
    priority => ... # ["normal", "low"] (optional)
    source_type_name => ... # ["nagios", "hudson", "jenkins", "user", "my apps", "feed", "chef", "puppet", "git", "bitbucket"] (optional), default: "my apps"
    tags => ... # array (optional), default: []
    text => ... # string (optional), default: "%{@message}"
    title => ... # string (optional), default: "Logstash event for %{@source_host}"
    type => ... # string (optional), default: ""
  }
}

Details

alert_type

  • Value can be any of: "info", "error", "warning", "success"
  • There is no default value for this setting.

api_key (required setting)

  • Value type is string
  • There is no default value for this setting.

date_happened

  • Value type is string
  • There is no default value for this setting.

dd_tags

  • Value type is array
  • There is no default value for this setting.

exclude_tags

  • Value type is array
  • Default value is []

fields

  • Value type is array
  • Default value is []

priority

  • Value can be any of: "normal", "low"
  • There is no default value for this setting.

source_type_name

  • Value can be any of: "nagios", "hudson", "jenkins", "user", "my apps", "feed", "chef", "puppet", "git", "bitbucket"
  • Default value is "my apps"

tags

  • Value type is array
  • Default value is []

text

  • Value type is string
  • Default value is "%{@message}"

title

  • Value type is string
  • Default value is "Logstash event for %{@source_host}"

type

  • Value type is string
  • Default value is ""
