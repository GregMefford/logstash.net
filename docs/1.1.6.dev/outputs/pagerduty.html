pagerduty

Status: experimental

PagerDuty output Send specific events to PagerDuty for alerting

Synopsis

This is what it might look like in your config file: 
output {
  pagerduty {
    description => ... # string (optional), default: "Logstash event for %{@source_host}"
    details => ... # hash (optional), default: {"timestamp"=>"%{@timestamp}", "message"=>"%{@message}"}
    event_type => ... # ["trigger", "acknowledge", "resolve"] (optional), default: "trigger"
    exclude_tags => ... # array (optional), default: []
    fields => ... # array (optional), default: []
    incident_key => ... # string (optional), default: "logstash/%{@source_host}/%{@type}"
    pdurl => ... # string (optional), default: "https://events.pagerduty.com/generic/2010-04-15/create_event.json"
    service_key => ... # string (required)
    tags => ... # array (optional), default: []
    type => ... # string (optional), default: ""
  }
}

Details

description

  • Value type is string
  • Default value is "Logstash event for %{@source_host}"

details

  • Value type is hash
  • Default value is {"timestamp"=>"%{@timestamp}", "message"=>"%{@message}"}

event_type

  • Value can be any of: "trigger", "acknowledge", "resolve"
  • Default value is "trigger"

exclude_tags

  • Value type is array
  • Default value is []

fields

  • Value type is array
  • Default value is []

incident_key

  • Value type is string
  • Default value is "logstash/%{@source_host}/%{@type}"

pdurl

  • Value type is string
  • Default value is "https://events.pagerduty.com/generic/2010-04-15/create_event.json"

service_key (required setting)

  • Value type is string
  • There is no default value for this setting.

tags

  • Value type is array
  • Default value is []

type

  • Value type is string
  • Default value is ""
